# Контрольні запитання - Лабораторна робота №3

## 1. Що таке реляційні бази даних? Що таке СУБД? Які СУБД ви знаєте?

Реляційна база даних зберігає дані у таблицях, які можуть бути пов'язані між собою ключами. СУБД - це система керування базами даних. Приклади: SQLite, MS SQL Server, PostgreSQL, MySQL, Oracle Database.

## 2. Що позначає термін таблиця у реляційній БД? Які існують зв'язки у реляційній БД?

Таблиця - це структура з рядками та стовпцями, де рядок є записом, а стовпець є властивістю. Основні зв'язки: один-до-одного, один-до-багатьох, багато-до-багатьох.

## 3. Що таке ERD-діаграма?

ERD-діаграма показує сутності бази даних, їхні поля та зв'язки. Вона потрібна для проєктування й пояснення структури БД.

## 4. Що таке DbContext?

DbContext - це головний клас Entity Framework Core для роботи з базою даних. Він описує таблиці через DbSet, конфігурацію моделей і дозволяє читати, додавати, змінювати та видаляти дані.

## 5. Що таке зв'язки one-to-one, one-to-many, many-to-many?

One-to-one: одному запису відповідає один інший запис, наприклад Customer і CustomerProfile. One-to-many: один запис має багато пов'язаних, наприклад Customer і Orders. Many-to-many: багато записів пов'язані з багатьма, наприклад Products і Tags.

## 6. Як реалізувати one-to-one через Fluent API?

Приклад: `HasOne(c => c.Profile).WithOne(p => p.Customer).HasForeignKey<CustomerProfileModel>(p => p.CustomerId)`.

## 7. Приклад many-to-many у EF Core

У цій роботі зв'язок many-to-many реалізовано між `ProductModel` і `ProductTagModel` через проміжну таблицю `ProductProductTags`.

## 8. Data Annotations vs Fluent API

Data Annotations - це атрибути над властивостями класу. Fluent API - це конфігурація в методі `OnModelCreating`. Fluent API гнучкіший і краще підходить для складних зв'язків.

## 9. Як створюється та застосовується міграція?

Міграція створюється командою `dotnet ef migrations add InitialCreate`, а застосовується командою `dotnet ef database update`.

## 10. Мета проєкту Infrastructure

`Infrastructure` містить код доступу до даних: EF-моделі, DbContext, репозиторії, міграції та CRUD-сервіси, які працюють з базою даних.

## 11. Доменна модель vs модель бази даних

Доменна модель описує бізнес-логіку програми. Модель бази даних описує структуру збереження даних у таблицях і зв'язках.

## 12. Що вирішує Repository?

Repository відокремлює бізнес-логіку від конкретного способу доступу до даних. Код працює з інтерфейсом, а не напряму з DbContext.

## 13. Реляційні vs нереляційні БД

Реляційні БД зберігають дані у таблицях зі схемою і зв'язками. Нереляційні БД можуть зберігати документи, ключ-значення, графи або колонки й часто мають гнучкішу структуру.
